# 途灵 TripMind 智能旅行助手 - 宝塔部署指南

## 1. 服务器准备

### 1.1 服务器要求

- **操作系统**：CentOS 7.6+ / Ubuntu 18.04+ / Debian 9+
- **CPU**：2核及以上
- **内存**：4GB及以上
- **磁盘**：40GB及以上
- **网络**：公网IP，带宽1Mbps以上

### 1.2 服务器初始化

1. **更新系统**：
   ```bash
   # CentOS
   yum update -y
   
   # Ubuntu/Debian
   apt update && apt upgrade -y
   ```

2. **安装必要工具**：
   ```bash
   # CentOS
   yum install -y wget curl unzip
   
   # Ubuntu/Debian
   apt install -y wget curl unzip
   ```

## 2. 宝塔面板安装

### 2.1 安装宝塔面板

1. **执行安装命令**：
   ```bash
   # CentOS
   yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh
   
   # Ubuntu/Debian
   wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh
   ```

2. **安装完成后**：
   - 记录面板登录地址、用户名和密码
   - 访问面板地址，登录并完成初始化设置

### 2.2 宝塔面板配置

1. **修改面板端口**（可选）：
   - 登录宝塔面板
   - 进入「面板设置」
   - 修改面板端口（建议使用10000-65535之间的端口）
   - 保存设置

2. **添加安全组规则**：
   - 在服务器提供商的控制台中，添加安全组规则
   - 开放面板端口、Web服务端口（80、443）、后端API端口（8000）

## 3. 环境配置

### 3.1 安装必要软件

1. **登录宝塔面板**，进入「软件商店」

2. **安装以下软件**：
   - **Nginx**：Web服务器
   - **MySQL**：数据库
   - **Python项目管理器**：用于管理Python项目
   - **Node.js**：用于前端构建

### 3.2 数据库配置

1. **创建数据库**：
   - 进入「数据库」
   - 点击「添加数据库」
   - 填写数据库名称、用户名和密码
   - 点击「提交」

2. **导入SQL文件**：
   - 进入创建的数据库
   - 点击「导入」
   - 选择 `backend/travel2.sql` 文件
   - 点击「开始导入」

### 3.3 Python环境配置

1. **安装Python**：
   - 进入「Python项目管理器」
   - 点击「版本管理」
   - 安装 Python 3.8+ 版本

2. **创建虚拟环境**：
   - 点击「虚拟环境」
   - 点击「创建虚拟环境」
   - 填写环境名称，选择Python版本
   - 点击「创建」

## 4. 项目部署

### 4.1 上传项目文件

1. **进入「文件」管理**

2. **创建项目目录**：
   - 在 `/www/wwwroot/` 目录下创建 `tripmind` 目录
   - 在 `tripmind` 目录下创建 `frontend` 和 `backend` 目录

3. **上传项目文件**：
   - 将前端项目文件上传到 `/www/wwwroot/tripmind/frontend`
   - 将后端项目文件上传到 `/www/wwwroot/tripmind/backend`

### 4.2 后端部署

1. **安装依赖**：
   - 进入「Python项目管理器」
   - 点击「添加项目」
   - 填写项目信息：
     - 项目路径：`/www/wwwroot/tripmind/backend`
     - 项目名称：`tripmind-backend`
     - 运行方式：`uWSGI`
     - 端口：`8000`
     - 虚拟环境：选择之前创建的虚拟环境
   - 点击「提交」

2. **配置环境变量**：
   - 进入项目详情
   - 点击「环境变量」
   - 添加以下环境变量：
     ```
     DB_HOST=localhost
     DB_PORT=3306
     DB_USER=数据库用户名
     DB_PASSWORD=数据库密码
     DB_NAME=数据库名称
     SECRET_KEY=随机字符串
     ```
   - 点击「保存」

3. **修改配置文件**：
   - 编辑 `backend/.env` 文件
   - 填写数据库连接信息

4. **启动服务**：
   - 点击「启动」按钮
   - 检查服务状态是否正常

### 4.3 前端部署

1. **安装依赖**：
   - 进入「终端」
   - 执行命令：
     ```bash
     cd /www/wwwroot/tripmind/frontend
     npm install
     ```

2. **构建项目**：
   - 执行命令：
     ```bash
     npm run build
     ```

3. **配置Nginx**：
   - 进入「网站」
   - 点击「添加站点」
   - 填写站点信息：
     - 域名：填写您的域名
     - 根目录：`/www/wwwroot/tripmind/frontend/dist`
     - PHP版本：纯静态
   - 点击「提交」

4. **配置反向代理**：
   - 进入站点详情
   - 点击「反向代理」
   - 点击「添加反向代理」
   - 填写代理信息：
     - 代理名称：`api`
     - 目标URL：`http://127.0.0.1:8000`
     - 发送域名：`$host`
     - 路径：`/api`
   - 点击「提交」

5. **配置SSL**（可选）：
   - 进入站点详情
   - 点击「SSL」
   - 选择「Let's Encrypt」
   - 填写邮箱
   - 点击「申请」
   - 开启「强制HTTPS」

## 5. 系统配置

### 5.1 后端配置

1. **修改配置文件**：
   - 编辑 `backend/config/development.yml` 文件
   - 填写正确的配置信息

2. **初始化数据库**：
   - 进入「终端」
   - 执行命令：
     ```bash
     cd /www/wwwroot/tripmind/backend
     python tools/init_db.py
     ```

### 5.2 前端配置

1. **修改API地址**：
   - 编辑 `frontend/.env` 文件
   - 设置 `VITE_API_BASE` 为您的API地址

2. **重新构建**：
   - 执行命令：
     ```bash
     cd /www/wwwroot/tripmind/frontend
     npm run build
     ```

## 6. 服务管理

### 6.1 启动服务

1. **启动后端服务**：
   - 进入「Python项目管理器」
   - 找到 `tripmind-backend` 项目
   - 点击「启动」

2. **启动前端服务**：
   - 前端服务由Nginx自动管理，无需单独启动

### 6.2 停止服务

1. **停止后端服务**：
   - 进入「Python项目管理器」
   - 找到 `tripmind-backend` 项目
   - 点击「停止」

2. **停止前端服务**：
   - 进入「网站」
   - 找到前端站点
   - 点击「停止」

### 6.3 重启服务

1. **重启后端服务**：
   - 进入「Python项目管理器」
   - 找到 `tripmind-backend` 项目
   - 点击「重启」

2. **重启前端服务**：
   - 进入「网站」
   - 找到前端站点
   - 点击「重启」

## 8. 常见问题

### 8.1 服务启动失败

**问题**：后端服务启动失败

**解决方法**：
1. 检查数据库连接信息是否正确
2. 检查依赖是否安装完整
3. 查看日志文件，分析错误原因
4. 尝试重新安装依赖

### 8.2 前端无法访问API

**问题**：前端页面无法访问后端API

**解决方法**：
1. 检查反向代理配置是否正确
2. 检查后端服务是否运行正常
3. 检查防火墙是否开放端口
4. 检查API地址配置是否正确

### 8.3 数据库连接失败

**问题**：后端无法连接数据库

**解决方法**：
1. 检查数据库连接信息是否正确
2. 检查数据库服务是否运行正常
3. 检查数据库用户权限是否正确
4. 尝试重启数据库服务

### 8.4 前端页面空白

**问题**：前端页面显示空白

**解决方法**：
1. 检查前端构建是否成功
2. 检查Nginx配置是否正确
3. 检查浏览器控制台是否有错误
4. 尝试重新构建前端项目

## 9. 安全配置

### 9.1 服务器安全

1. **修改SSH端口**：
   - 进入「安全」
   - 点击「SSH端口」
   - 修改默认端口
   - 点击「保存」

2. **开启防火墙**：
   - 进入「安全」
   - 开启「防火墙」
   - 添加必要的端口规则

3. **设置登录保护**：
   - 进入「面板设置」
   - 开启「登录保护」
   - 设置登录限制

### 9.2 网站安全

1. **配置HTTPS**：
   - 为前端和后端服务配置SSL证书
   - 开启强制HTTPS

2. **设置防盗链**：
   - 进入前端站点详情
   - 点击「防盗链」
   - 开启防盗链保护

3. **配置WAF**：
   - 进入「安全」
   - 开启「Web应用防火墙」
   - 配置防护规则
